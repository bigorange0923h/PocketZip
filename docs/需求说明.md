# PocketUnzip Windows 解压工具需求说明文档

## 1. 项目背景

本项目计划实现一款 Windows 桌面解压工具，支持用户快速解压常见压缩包格式，并提供密码记录、历史记录、批量解压等能力。

项目第一阶段采用 `7z.exe` 作为解压内核，通过桌面应用封装交互能力，降低开发复杂度，提高压缩格式兼容性。

## 2. 项目目标

实现一个轻量、易用、支持密码记忆的 Windows 解压程序。

核心目标：

- 支持常见压缩包格式解压
- 支持加密压缩包密码输入
- 支持成功密码记录
- 支持下次自动尝试历史密码
- 支持解压历史查看
- 支持解压日志展示
- 支持自动按压缩包名称创建目录

## 3. 目标用户

- Windows 普通用户
- 经常处理压缩包的办公用户
- 经常下载带密码压缩包的用户
- 需要批量解压文件的用户

## 4. MVP 功能范围

### 4.1 文件选择

用户可以通过以下方式添加压缩包：

- 点击选择文件
- 拖拽压缩包到窗口
- 后续支持右键菜单打开

### 4.2 输出目录选择

用户可以选择解压目录。

支持配置：

- 解压到当前目录
- 解压到指定目录
- 自动创建同名文件夹

示例：

```text
D:\Downloads\test.zip
解压到：
D:\Downloads\test\
```

### 4.3 解压功能

调用内置 `7z.exe` 执行解压。

支持格式以 7-Zip 实际支持能力为准，包括但不限于：

- zip
- 7z
- rar
- tar
- gz
- bz2
- xz
- iso
- cab

### 4.4 密码输入

当压缩包需要密码时：

- 用户手动输入密码
- 支持显示 / 隐藏密码
- 支持记住密码
- 支持取消解压

### 4.5 密码记录

解压成功后，如果用户选择保存密码，则记录该压缩包对应密码。

密码不得明文保存。

密码保存策略：

- 使用 Windows DPAPI 加密
- SQLite 只保存加密后的密码
- 支持按文件路径、文件名、文件哈希匹配密码

### 4.6 自动尝试密码

再次解压加密压缩包时，系统按以下顺序尝试密码：

```text
1. 当前压缩包路径对应密码
2. 当前压缩包文件名对应密码
3. 历史常用密码
4. 用户手动输入
```

### 4.7 解压历史

记录每次解压行为：

- 压缩包路径
- 输出目录
- 是否成功
- 是否使用密码
- 错误信息
- 解压时间

### 4.8 解压日志

界面展示实时日志：

- 当前执行命令
- 解压进度
- 成功文件
- 错误信息
- 密码错误提示

## 5. 非 MVP 功能

后续版本支持：

- 批量解压
- 右键菜单集成
- 压缩包内容预览
- 解压前测试压缩包
- 密码库管理
- 自动清理重复密码
- 解压完成后打开目录
- 解压完成通知
- 主题切换
- 自动更新

## 6. 安全要求

### 6.1 密码安全

- 禁止明文保存密码
- 禁止在日志中打印明文密码
- 禁止在错误信息中暴露密码
- 密码加密使用 Windows DPAPI

### 6.2 命令执行安全

调用 `7z.exe` 时必须使用参数数组，不允许拼接完整命令字符串执行。

推荐方式：

```go
exec.Command("7z.exe", "x", archivePath, "-o"+outputDir, "-p"+password, "-y")
```

不推荐：

```go
exec.Command("cmd", "/C", commandString)
```

### 6.3 路径安全

需要处理：

- 路径空格
- 中文路径
- 特殊字符路径
- 输出目录不存在
- 文件覆盖提示
- 解压路径穿越风险

## 7. 成功标准

MVP 完成标准：

- 可以选择压缩包并成功解压
- 可以解压带密码压缩包
- 密码错误时能明确提示
- 解压成功后可以保存密码
- 下次可以自动使用已保存密码
- 可以查看解压历史
- 可以查看解压日志
- 程序可打包为 Windows exe
