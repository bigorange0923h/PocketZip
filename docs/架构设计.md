# PocketZip 架构设计

## 1. 总体架构

PocketZip 使用 Go + Wails 构建 Windows 桌面应用。Go 负责业务逻辑、7z.exe 调用、SQLite 存储、DPAPI 加密；前端负责文件选择、拖拽、日志展示、历史展示和密码输入。

```text
Frontend UI
   ↓
Wails Bridge
   ↓
App Service
   ↓
Archive Service ── 7z.exe
Password Service ── DPAPI + SQLite
History Service ── SQLite
Config Service ── local config
```

## 2. 核心模块

### 2.1 App Service

应用编排层，负责连接前端请求和后端能力。

主要职责：

- 接收解压请求
- 计算输出目录
- 调用密码匹配流程
- 调用 Archive Service 解压
- 写入历史记录
- 推送实时日志到前端

### 2.2 Archive Service

封装 `7z.exe` 调用。

主要职责：

- 构造安全参数数组
- 执行解压命令
- 解析 7z 输出日志
- 判断密码错误、文件损坏、路径错误等异常
- 禁止通过 `cmd /C` 拼接命令执行

### 2.3 Password Service

负责密码保存和自动匹配。

匹配顺序：

1. archive path
2. file name
3. file hash
4. common password
5. manual input

保存要求：

- 明文密码只存在内存中
- SQLite 只保存 DPAPI 加密后的密文
- 日志中不输出密码

### 2.4 History Service

记录解压历史。

字段建议：

- id
- archive_path
- output_dir
- success
- used_password
- error_message
- created_at

### 2.5 Security Service

安全工具模块。

主要职责：

- DPAPI 加密
- DPAPI 解密
- 日志脱敏
- 路径合法性检查

## 3. 解压流程

```text
用户选择压缩包
   ↓
计算输出目录
   ↓
尝试无密码解压
   ↓
如果需要密码，读取历史密码候选
   ↓
逐个尝试密码
   ↓
仍失败则弹出密码输入框
   ↓
解压成功
   ↓
按用户选择保存密码
   ↓
写入历史记录
   ↓
展示完成结果
```

## 4. 数据库设计

### 4.1 password_records

```sql
CREATE TABLE password_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    archive_path TEXT,
    archive_name TEXT,
    archive_hash TEXT,
    encrypted_password BLOB NOT NULL,
    success_count INTEGER DEFAULT 1,
    last_used_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4.2 extract_history

```sql
CREATE TABLE extract_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    archive_path TEXT NOT NULL,
    output_dir TEXT NOT NULL,
    success INTEGER NOT NULL,
    used_password INTEGER DEFAULT 0,
    error_message TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 4.3 app_config

```sql
CREATE TABLE app_config (
    key TEXT PRIMARY KEY,
    value TEXT,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 5. 7z 命令设计

推荐参数：

```go
args := []string{"x", archivePath, "-o" + outputDir, "-y"}
if password != "" {
    args = append(args, "-p"+password)
}
cmd := exec.Command(sevenZipPath, args...)
```

日志展示时必须脱敏：

```text
7z.exe x test.zip -oD:\Downloads\test -p****** -y
```

## 6. 风险点

- 不同格式的 7z 输出不完全一致
- RAR5 支持能力取决于 7z 版本
- 密码错误和文件损坏有时需要结合 exit code 与 stderr 判断
- 解压路径穿越需要额外校验
- 批量解压时要控制并发，避免磁盘 IO 爆掉
