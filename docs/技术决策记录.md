# PocketUnzip 技术决策记录

## ADR-001：解压内核选择 7z.exe

### 背景

Windows 解压工具需要支持多种压缩格式。如果直接集成各类格式 SDK，开发成本和兼容性风险较高。

### 决策

MVP 阶段采用 `7z.exe` 作为解压内核，通过 Go 后端调用命令行完成解压。

### 理由

- 支持格式多
- 开发复杂度低
- 行为稳定
- 便于快速完成 MVP
- 后续可替换为 7-Zip SDK 或 libarchive

### 代价

- 需要随应用分发 7z.exe / 7z.dll
- 日志解析依赖命令行输出
- 进度解析不如 SDK 精准

## ADR-002：密码使用 Windows DPAPI 加密

### 决策

密码保存时使用 Windows DPAPI 加密，SQLite 只保存密文。

### 理由

- 不需要用户额外设置主密码
- 与 Windows 用户账户绑定
- 适合个人桌面工具

### 注意

- 换 Windows 账户后可能无法解密旧密码
- 跨设备同步密码需要另行设计

## ADR-003：数据库选择 SQLite

### 决策

本地数据使用 SQLite。

### 理由

- 免安装
- 单文件数据库
- 适合桌面应用
- 足够支撑历史记录、密码记录、配置数据
